MSSQL – SSMS Bağlantı Kurulum Kılavuzu

EBA platformunu MSSQL Server'a bağlamak için aşağıdaki adımları sırasıyla uygulayın

🔴 MSSQL Bağlantısı YOK – Yerel DB kullanılıyor Hata: Login failed for user 'sa' — Mixed Mode Authentication kapalı olabilir
1
SSMS'i Aç ve SQL Server'a Bağlan
Windows Authentication ile giriş yap
  • 🔍
    Başlat menüsünden ara: SQL Server Management Studio
  • 🖥️
    Bağlan ekranında:
    Server Type
    Database Engine
    Server Name
    localhost veya .
    Authentication
    Windows Authentication
    Connect
    Connect butonuna bas
2
SQL Server Authentication (Mixed Mode) Aktif Et
Bu adım olmadan sa ile bağlantı çalışmaz
Neden gerekli? EBA, sa hesabı ile bağlanmaya çalışıyor. SQL Server varsayılan olarak sadece Windows Authentication kullanır. Mixed Mode'u açarak SQL hesaplarına izin vermeniz gerekiyor.
  • 🖱️
    SSMS'de sol tarafta Object Explorer'da sunucu adına (en üst satır) sağ tıkla → Properties
  • 🔒
    Açılan pencerede sol menüden Security sekmesine tıkla
  • ✅
    Server authentication bölümünde "SQL Server and Windows Authentication mode" seçeneğini seç (yani Mixed Mode)
  • 💾
    OK butonuna bas
  • ⚠️
    Önemli: Bir uyarı çıkacak ve SQL Server'ı yeniden başlatmanız istenecek — bunu kabul et
3
sa Hesabını Aktif Et ve Şifre Belirle
sa hesabı varsayılan olarak devre dışı olabilir
EBA'nın bağlanmak istediği hesap: sa — şifre: 123456
📋 Yöntem A: SSMS ile (görsel)
  • 📂
    SSMS'de: Security → Logins → sa üzerine çift tıkla
  • 🔑
    General sekmesinde şifreyi 123456 olarak gir, tekrar gir
  • 🔓
    Status sekmesine geç → Login: Enabled seç
  • 💾
    OK'a bas
📋 Yöntem B: SQL komutu ile (daha hızlı)

SSMS'de New Query aç, aşağıdaki kodu yapıştır ve F5 ile çalıştır:

📋 Kopyala ALTER LOGIN sa WITH PASSWORD = '123456', CHECK_POLICY = OFF, CHECK_EXPIRATION = OFF; ALTER LOGIN sa ENABLE; GO
4
SQL Server Servisini Yeniden Başlat
Mixed Mode değişikliği ancak yeniden başlatmadan sonra geçerli olur
📋 Yöntem A: SSMS ile
  • 🖱️
    SSMS'de sunucu adına sağ tıkla → Restart
  • ✅
    Çıkan onay kutusuna Yes de
📋 Yöntem B: Windows PowerShell ile (Yönetici olarak aç)
📋 Kopyala Restart-Service -Name 'MSSQLSERVER' -Force
Servis yeniden başlayınca SSMS'de bağlantı geçici olarak kesilecek. Bu normaldir — 10-20 saniye bekle.
5
EBA_DB Veritabanını Oluştur
EBA sunucusu bu veritabanı içine tablolar oluşturacak

SSMS'de yeni New Query açıp çalıştır:

📋 Kopyala IF NOT EXISTS (SELECT * FROM sys.databases WHERE name = 'EBA_DB') BEGIN CREATE DATABASE EBA_DB; PRINT 'EBA_DB oluşturuldu!'; END ELSE PRINT 'EBA_DB zaten mevcut.'; GO
